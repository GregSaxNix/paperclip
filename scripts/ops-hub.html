📎 Paperclip — what it is

Problem it solves: Running a small fleet of specialised AI agents (CEO, CTO, Researcher, Council members, etc.) with separate roles, prompts, and model assignments — and being able to swap models or run them in fallback chains when one is rate-limited.

What it does: Hosts agent definitions, manages issues/comments between agents, runs heartbeats, tracks usage, and provides a chat UI to talk to any agent. 22 positions configured. Currently 1 company (Life Admin).

Where it lives: D:\paperclip — server on port 3100 (auto-starts via Task Scheduler).

📊 Agent Matrix & 🧠 LLM Matrix — what they are

Agent Matrix: Live view of all 22 positions — which model each one runs on, fallback chains, usage meters, active status. Use it to spot when an agent is at risk of hitting a quota limit.

LLM Matrix: All available LLMs scored on tasks (research, code, write, etc.), with pricing. Recommends which model fits which position. Use it weekly to decide if you should re-assign a model to a position.

Refresh: python scripts\position_manager.py dashboard for Agent Matrix; python scripts\update_llm_matrix.py for LLM Matrix.

🎬 YouTube Brain — what it is

Problem it solves: You watch a lot of YouTube. You can never remember which video had that specific point about X. Standard YouTube search doesn't work because it's not your library, and YouTube's history is unsearchable.

What it does: Indexes every video you've watched (~9,760 currently), pulls the transcript, classifies into topics, lets you full-text search across the lot. Default landing view shows your watch history sorted most-recent first; layer on liked/topic/channel/year filters in any combination.

Where it lives: D:\youtube-brain — server on port 8789. SQLite database with FTS index. Local LLM (Ollama) for classification — no cloud.

📚 Second Brain — what it is

Problem it solves: YouTube Brain stores raw videos. But you also want a curated wiki of concepts you care about and creators you trust — built from those transcripts plus other sources (PDFs, URLs, notes).

What it does: Karpathy-style "compile once, keep current" knowledge wiki. Concept pages like ai-agents.md, entity pages like matthew-berman.md. Ingests new YouTube transcripts daily, appends "Update:" blocks to the relevant pages. Obsidian-compatible.

Where it lives: D:\second-brain — server on port 8790. Different from Assistant OS — Second Brain is world-knowledge; Assistant OS is self-knowledge about Greg.

🧠 Assistant OS — what it is

Problem it solves: You use multiple AI tools (Claude Code, Cursor, ChatGPT). Each one starts knowing nothing about you. Re-explaining your style, preferences, Shine context, project state every time is wasted effort. And if you switch tools, that knowledge doesn't follow.

What it does: Single Git-versioned Markdown store (D:\assistant-os\canonical\) of identity, preferences, working style, current state, Shine domain knowledge. Compilers generate ready-to-use config for Claude Code (CLAUDE.md + .claude/rules/) and Cursor (AGENTS.md + .cursor/rules/*.mdc). Drop those into any project and the assistant immediately knows you. ChatGPT/OpenClaw/PAM/MCP exporters deferred to Phase 2.

Inbox — Dismiss failed run notifications
ℹ️ When will failed runs appear, and what to do

Failed runs appear in the Paperclip inbox when an agent heartbeat fails — for example, when a usage limit is hit or a model is temporarily unavailable. Once the underlying issue is resolved (limit resets, fallback kicks in), the old failures are just noise.

They will recur each time an agent hits a quota or adapter error — especially once heartbeats are enabled. The script below dismisses whatever is currently showing. You'll need to re-run it after new failures appear.

To dismiss: Open Paperclip in Chrome → F12 → Console tab → paste the script below → press Enter → refresh the page.

// Dismiss ALL current failed runs from the inbox:
const key = "paperclip:inbox:dismissed";
fetch("/api/companies/505ab906-66b5-4400-b131-96b8aee91c5d/heartbeat-runs?limit=50")
  .then(r => r.json())
  .then(runs => {
    const arr = Array.isArray(runs) ? runs : (runs.runs || runs.data || []);
    const ids = arr.filter(r => r.status === "failed" || r.status === "timed_out").map(r => "run:" + r.id);
    const existing = JSON.parse(localStorage.getItem(key) || "[]");
    localStorage.setItem(key, JSON.stringify([...new Set([...existing, ...ids])]));
    console.log("Dismissed", ids.length, "failed runs. Refresh the page.");
  });

This version auto-fetches all current failed runs — no need to update the script manually each time.

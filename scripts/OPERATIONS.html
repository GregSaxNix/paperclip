Accessing Paperclip from Your Laptop

Tailscale Funnel is already set up. Nothing to install on the laptop.

  1. Make sure the desktop is on and not asleep
  2. On your laptop browser, go to:
https://greg-shinedesktop.tail4ef50.ts.net

Full Paperclip UI — works anywhere in the world, no VPN needed.

If Paperclip won't load, the server may not be running. Use Remote Desktop (below) to restart it, or reboot the desktop — it auto-starts on boot.

Remote Desktop (to run terminal commands remotely)

One-time setup on the desktop

Search "Remote Desktop settings" → turn on "Enable Remote Desktop"

Get the Tailscale IP (run once on desktop, save the result)

tailscale ip -4

Connect from laptop

  • Windows laptop: Open "Remote Desktop Connection" → enter the Tailscale IP
  • Mac laptop: Install "Microsoft Remote Desktop" from App Store → New PC → enter Tailscale IP
  • Log in with your Windows username and password

Auto-Monitor (runs while you're away)

Installed as a Windows Task Scheduler job called LifeAdmin-Monitor. Runs every 2 hours automatically — no action needed.

What it does

  1. Checks Claude usage
  2. If All Models or Session hits 90% → subs all claude_local agents to fallback1
  3. When usage resets below 90% → restores all agents back to primary
  4. Logs all actions to D:\paperclip\monitor.log

Check the log

python D:\paperclip\scripts\auto_monitor.py --log

Run it manually

python D:\paperclip\scripts\auto_monitor.py

Inbox — Dismissing Failed Run Notifications

Failed run notifications appear when an agent heartbeat fails — quota hits, model unavailable, etc. Once the underlying issue is fixed, old failures are just noise. They will recur whenever agents hit limits, especially once heartbeats are enabled.

To dismiss: Open Paperclip in Chrome → F12 → Console tab → paste this → press Enter → refresh the page:

// Auto-dismiss all current failed runs:
const key = "paperclip:inbox:dismissed";
fetch("/api/companies/505ab906-66b5-4400-b131-96b8aee91c5d/heartbeat-runs?limit=50")
  .then(r => r.json())
  .then(runs => {
    const arr = Array.isArray(runs) ? runs : (runs.runs || runs.data || []);
    const ids = arr
      .filter(r => r.status === "failed" || r.status === "timed_out")
      .map(r => "run:" + r.id);
    const existing = JSON.parse(localStorage.getItem(key) || "[]");
    localStorage.setItem(key, JSON.stringify([...new Set([...existing, ...ids])]));
    console.log("Dismissed", ids.length, "failed runs. Refresh the page.");
  });
This script auto-fetches current failed runs — no need to update it. Run it whenever the inbox fills up with old failures.

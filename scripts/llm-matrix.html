How the Hiring Algorithm Works

Each position has a task weight profile in llm-matrix-data.json (e.g. CEO = 30% reasoning, 25% research, 20% writing...). The algorithm scores every model by computing sum(task_score × weight) across all task dimensions. The highest-scoring model becomes the recommended Primary; next two become Fallback 1 and Fallback 2. Emergency is always gemma4:26b — it is the always-available local fallback regardless of score.

Local-only positions (Doc, Freud, Council Local) only consider models where local_only: true. If a better model is released or pricing drops significantly, re-run --hire to see if lineup changes are warranted.

Greg approves all changes — the algorithm recommends, Greg decides, position_manager.py sub applies.
